A response to
CSP is Responsive Design

Written by Logan Campbell on the 3rd of August 2013.

Recently David Nolan proposed a novel three part design pattern in a post titled CSP is Responsive Design. He demonstrates how a combination of Event Stream Processing and CSP can be used to simplify UI programming. The three parts he proposes are:

  1. Event stream processing
  2. Event stream coordination
  3. Interface representation

This caught my attention as recently I've been thinking hard about using event stream processing for UIs. However the concept of event stream coordination was new to me. No strict definition is given, rather he illustrates by way of examples.

The coordination functions selector and highlighter take and return core.async channels.
