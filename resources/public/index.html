A response to
CSP is Responsive Design

Written by Logan Campbell on the 3rd of August 2013.

Recently David Nolan has written about how a combination of event stream processing and Communicating Sequencial processes can be used to simplify UI programming. He proposes a novel three part architecture consisting of:

  1. Event stream processing
  2. Event stream coordination
  3. Interface representation

I'm quite taken with stream processing, so much so that I'm writing a ClojureScript library that enables it. Interface representation is a brilliant idea and I wish I'd thought of it before. However stream coordination was new to me and it is the main focus of this response.

Stream Coordination Examples

Nolan gives no strict definition for stream coordination, instead he illustrates with examples. It is possible that I've missunderstood the concept and that I'm mistaking shortcomings of toy examples for shortcomings of the concept. However I feel the examples are more complex, and less functional, than raw stream processing. So I'm left feeling that stream coordination should be avoided.

The coordination functions selector and highlighter take and return core.async channels. This is great as it means these processes don't care where the events come from or end up. Composing them extends the functionality of the UI. But I feel there are some drawbacks to this approach:

  1. Neither function is pure. They pull values out of the channel. This both mutates the chanel (removing the value) and means we can not determine the return values purely from function arguments.
  2. Recognition of events and processing are handled in the same function. A simpler design would split these responsabilities.
  3. Explicit flow control (loop/recur) and event emition (>!) are required. If higher-order functions were used both of these things could be eliminated.
  4. By only passing along unknown events the functions must assume all responsability for those events which they process. This is less flexible than allowing for multiple consumers of each channel.

Raw stream processing
